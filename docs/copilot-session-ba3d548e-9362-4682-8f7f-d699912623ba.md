Any contributor can build and run the API locally using Docker and Python — no cloud account required.

- Python 3.10+
- Java 11+ (required by the OpenAPI code generator)

```bash
scripts/setup-openapi-generator.sh
```

```bash
docker compose --env-file ./config/.env.local up -d
```

```bash
cd api

pip install -r requirements_dev.txt
```

```bash
scripts/api-start.sh
```

## Deploying the API

### Prerequisites

- GCP service account with the required roles (see `infra/`)
- GitHub secrets configured: `GCP_MOBILITY_FEEDS_SA_KEY`, `OAUTH2_CLIENT_ID`, `OAUTH2_CLIENT_SECRET`, `OP_SERVICE_ACCOUNT_TOKEN`
- A Docker image published to GHCR: `ghcr.io/mobilitydata/mobility-feed-api:{version}`

### How to deploy

1. Go to **Actions** → `Deploy Feeds API - Dev` (or QA/Prod)
2. Click **Run workflow**
3. Provide the `FEED_API_IMAGE_VERSION` — this is the GHCR tag to deploy (e.g., `v1.2.3` or `sha-abc123`)
4. The workflow will:
   - Ensure GCP Artifact Registry exists (Terraform)
   - Pull the image from GHCR and push it to GCP Artifact Registry
   - Deploy to Cloud Run via Terraform
